## Introduction

DeskHub is a web-based Windows desktop remote control tool. It turns your PC into a LAN service node — any device on the same Wi-Fi can control it through a browser.

### Why This Project?

Ever been lying in bed wanting to control music on your PC, or too lazy to get up and shut it down? DeskHub was born to solve these everyday pain points. No app installation needed on your phone — just scan and go.

### Core Principles

- **Zero Client**: No app installation on mobile — the browser is the entry point
- **LAN First**: All data stays local, no cloud services involved — your privacy is safe
- **Plug & Play**: Double-click the launcher and you're set — auto environment check, dependency install, QR code generation
- **Brutalist Aesthetics**: Pure black base + bright green/orange accents + grid texture — anything but boring

### How It Works

```
┌─────────────┐     Wi-Fi / LAN     ┌─────────────────┐
│ Phone Browser│ ◄──────────────────► │   PC (DeskHub)  │
│  Scan & Go   │    HTTP + WebSocket  │  Frontend :5173 │
└─────────────┘                      │  Backend  :3000 │
                                     └─────────────────┘
```

1. Run DeskHub on your PC (Node.js backend + React frontend)
2. Frontend dev server enables LAN access (Vite `host: true`)
3. Scan the QR code generated by the launcher, open in phone browser
4. Bidirectional communication via REST API and WebSocket
5. Backend invokes Windows system commands to perform actual operations (launch apps, open URLs, execute commands, etc.)

## Getting Started

### Prerequisites

| Dependency | Version | Notes |
|-----------|---------|-------|
| Node.js | >= v16 | LTS version recommended |
| Windows | 7+ | App launch, system commands depend on Windows |
| Default Audio Player | Any | System-associated audio player works (optional) |

### Installation

```bash
# Clone the repository
git clone https://github.com/your-username/desk_hub.git
cd desk_hub

# Install dependencies (npm workspaces installs all sub-packages)
npm install
```

### Launch

**Option 1: Launcher (Recommended)**

Double-click `launch-deskhub.bat` to enter the interactive menu. The launcher automatically:
- Checks Node.js environment
- Detects port conflicts
- Installs missing dependencies
- Starts frontend & backend services
- Generates QR code for phone access
- Opens browser

**Option 2: Command Line**

```bash
# Start both frontend and backend
npm run dev

# Or start separately
npm run dev:server   # Backend → http://localhost:3000
npm run dev:client   # Frontend → http://localhost:5173
```

### Access URLs

| Access Method | URL | Notes |
|--------------|-----|-------|
| Local | `http://localhost:5173/` | Direct access from PC browser |
| LAN | `http://YOUR_IP:5173/` | Phone/tablet access, same Wi-Fi required |

> The Vite dev server has `host: true` enabled, automatically listening on all network interfaces for LAN access.

## Feature Details

### App Launcher

Remotely launch desktop applications.

**Features:**
- Click card to launch app
- Right-click / long-press to edit
- Add new apps (supports browser upload of .exe / .lnk / .bat / .cmd files)
- Delete apps
- Custom icons (emoji or base64 image, image takes priority)
- Auto-extract filename as app name on upload
- Green edit button and red delete button appear on hover

**Actions:**
| Action | Trigger |
|--------|---------|
| Launch app | Click card |
| Edit app | Right-click card / click top-left edit button |
| Delete app | Click top-right delete button |
| Add app | Click "Add App" button at page top |

### Web Quick Access

One-click to open websites with bookmark management.

**Features:**
- Click card to open URL in default browser
- Quick URL input (auto-prepend https://)
- Add / edit / delete bookmarks
- Custom icons (emoji or base64 image)
- Orange edit button and red delete button on hover
- Press Enter to quickly open URL from input

**Smart Browser Launch:**
- Browser running → open URL directly
- Browser not running → launch default browser, wait 1.5s, then open target URL
- Uses `cmd /c start` to avoid process blocking

### System Commands

Remotely execute system commands with preset and custom support.

**Preset Commands:**

| Command | Icon | Color | Description |
|---------|------|-------|-------------|
| `shutdown` | Power | Red | Shut down |
| `restart` | RotateCcw | Orange | Restart |
| `hibernate` | Moon | Green | Hibernate |
| `lock` | Lock | Orange | Lock screen |

**Features:**
- One-click preset execution, dangerous operations require confirmation
- Custom command input, press Enter to execute
- Command output streamed to frontend via WebSocket
- Custom commands can be marked as dangerous (warning icon)

**Confirmation Dialog:**
Dangerous commands (shutdown/restart etc.) show a confirmation dialog before execution. The dialog uses `createPortal` to render at `document.body`, avoiding parent `transform` interference with `fixed` positioning.

### Local Music Search

Scan music files on PC, fuzzy search, open with default audio player.

**Scan Scope:**
- `~/Music`, `~/Downloads`, `~/Desktop`
- `D:/Music`, `D:/音乐`, `D:/Downloads` (D/E/F drives)
- Max 4-level directory recursion
- Skips hidden dirs, `node_modules`, `$RECYCLE.BIN`

**Supported Audio Formats:**
`.mp3` `.flac` `.wav` `.aac` `.ogg` `.wma` `.m4a` `.ape` `.alac` `.opus`

**Search Algorithm (Scoring):**

| Match Type | Score | Example |
|-----------|-------|---------|
| Exact match | +100 | Search "hello" → filename "hello" |
| Prefix match | +50 | Search "hel" → filename "hello" |
| Substring match | +30 | Search "ell" → filename "hello" |
| Directory match | +10 | Search "rock" → directory "rock" |
| Character order | +5/char | Search "hlo" → filename "hello" (h→l→o in order) |

- Space-separated words, all must match
- Results sorted by score descending
- 150ms debounced real-time search

**Open Method:**
1. Try `cmd /c start` first (uses file association)
2. On failure, find an installed audio player and launch directly
3. 1-second timeout auto-resolve as success

**Format Color Coding:**
- FLAC / APE → Green
- MP3 / M4A → Orange
- Others → Gray

### System Info

Display static hardware information (not real-time monitoring).

**Display Items:**
- OS (version, architecture, hostname)
- Processor (model, cores, frequency)
- Memory (total)
- GPU (model, VRAM)
- Disks (filesystem, total, used)
- Network adapters (interface, IP, MAC)

> Data collected via `systeminformation` library, fetched once on page load, not refreshed in real-time.

## Configuration

Edit `config.json` in the project root to customize:

```json
{
  "server": {
    "port": 3000
  },
  "apps": [
    {
      "name": "App Name",
      "path": "Shortcut path (.lnk) or executable path (.exe)",
      "icon": "emoji or base64 image"
    }
  ],
  "websites": [
    {
      "name": "Website Name",
      "url": "https://example.com",
      "icon": "emoji or base64 image"
    }
  ],
  "commands": {
    "presets": ["shutdown", "restart", "hibernate", "lock"],
    "custom": [
      {
        "name": "Custom Command Name",
        "cmd": "Command to execute",
        "dangerous": false
      }
    ]
  },
  "music": {
    "defaultLimit": 20
  }
}
```

### Field Reference

| Field | Type | Description |
|-------|------|-------------|
| `server.port` | number | Backend server port, default 3000 |
| `apps` | array | App list, each item has name / path / icon |
| `websites` | array | Bookmark list, each item has name / url / icon |
| `commands.presets` | string[] | Enabled preset command keys |
| `commands.custom` | array | Custom command list, each item has name / cmd / dangerous |
| `music.defaultLimit` | number | Default music search result limit |

### Icon Format

Icons support two formats, **image takes priority over emoji**:

| Format | Example | Description |
|--------|---------|-------------|
| Emoji | `"💻"` | Direct emoji character |
| Base64 Image | `"data:image/png;base64,..."` | Base64-encoded image data |

### App Path

App paths support the following formats:

| Format | Example | Description |
|--------|---------|-------------|
| Shortcut | `"C:\\Users\\...\\app.lnk"` | .lnk shortcut recommended |
| Executable | `"C:\\Program Files\\app\\app.exe"` | Direct .exe path |
| CLI Command | `"code"` | Command in PATH |
| Upload | Via browser upload | Auto-saved to `data/shortcuts/` |

## API Reference

Backend runs at `http://localhost:3000/api/`, all endpoints return JSON.

### Apps

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/apps` | Get app list |

| POST | `/api/apps/upload` | Upload shortcut file |
| POST | `/api/config/apps` | Add app |
| PUT | `/api/config/apps` | Update app list |
| DELETE | `/api/config/apps/:name` | Delete app |

### Websites

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/web/bookmarks` | Get bookmark list |
| POST | `/api/web/open` | Open URL (body: `{ url }`) |
| POST | `/api/config/websites` | Add bookmark |
| PUT | `/api/config/websites` | Update bookmark list |
| DELETE | `/api/config/websites/:name` | Delete bookmark |

### Commands

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/commands` | Get command list (presets + custom) |
| POST | `/api/commands/execute` | Execute command (body: `{ name, cmd? }`) |

### System

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/system/info` | Get system hardware info |

### Music

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/music/scan` | Scan local music files |
| GET | `/api/music/status` | Get index status |
| GET | `/api/music/search?keyword=xxx&limit=50` | Search local music |
| POST | `/api/music/open` | Open with NetEase Cloud Music (body: `{ path }`) |

### WebSocket

Connection URL: `ws://localhost:3000/ws`

**Message Types:**

| Type | Direction | Description |
|------|-----------|-------------|
| `ping` / `pong` | Bidirectional | Heartbeat |
| `_connected` | Server→Client | Connection established |
| `_disconnected` | Server→Client | Connection lost |
| `command:output` | Server→Client | Real-time command output |

### Startup Flow

```
Launcher
  ├── Detect expired logs → Ask whether to clean
  ├── Show service status
  └── Interactive Menu
        ├── [1] Start DeskHub
        │     ├── Check Node.js environment
        │     ├── Detect port conflicts
        │     ├── Verify dependencies
        │     ├── Start backend service
        │     ├── Start frontend service
        │     ├── Wait for ports ready (max 60s)
        │     ├── Show access URLs + generate QR code
        │     └── Auto-open browser
        ├── [L] Clean log files
        │     ├── Show file list (size / age)
        │     ├── [1] Clean expired files (7+ days)
        │     └── [2] Clean all files
        └── [0] Exit
```

### Running Menu

When services are already running:

| Option | Description |
|--------|-------------|
| [R] | Restart services (stop then start) |
| [S] | Stop services (kill processes + clean orphans) |
| [I] | Show access URLs + generate QR code |
| [O] | Open in browser |
| [L] | Clean log files |
| [C] | Continue running (no action) |
| [Q] | Quit script (services keep running in background) |

### Feature Details

**IP Detection:**
- Auto-excludes virtual adapters (VMware / VirtualBox / Hyper-V / WSL / Docker etc.)
- Prefers Ethernet, then Wi-Fi / WLAN
- Ensures LAN address is from a real physical adapter

**QR Code Generation:**
- Generated via `api.qrserver.com` online API
- Auto-saved as PNG and opened
- Falls back to URL text box when offline

**Process Management:**
- Uses `netstat -ano` to find PIDs listening on specific ports
- Distinguishes DeskHub processes from external ones
- Cleans up orphan tsx watch processes on stop
- Force-kills all node processes if ports remain occupied

**Log Management:**
- Logs saved in `.deskhub-logs/` directory
- Naming: `frontend-YYYYMMDD-HHmmss.log` / `backend-YYYYMMDD-HHmmss.log`
- QR images: `qr-YYYYMMDD-HHmmss.png`
- Detects expired files (7+ days) on startup, asks whether to clean
- Manual view and clean available in menu

### Project Conventions

| Convention | Description |
|-----------|-------------|
| EditModal Rendering | Must use `ReactDOM.createPortal` to `document.body` to avoid parent `transform` interfering with `fixed` positioning |

1. Create page component in `client/src/pages/`
2. Add route and navigation item in `client/src/App.tsx`
3. Add navigation entry in `client/src/components/Sidebar.tsx` and `Drawer.tsx`
4. Add corresponding API in `server/src/routes/index.ts`
5. Add type definitions in `shared/types/index.ts`

## FAQ

### Phone Can't Access

1. Confirm phone and PC are on the same Wi-Fi
2. Check if Windows Firewall allows ports 5173 and 3000
3. Verify the LAN IP shown by the launcher is from a physical adapter (not virtual)

### Font Loading Failed

The project uses `fonts.loli.net` (China mirror). If still failing, check network proxy settings.

### Browser Doesn't Auto-Open

The launcher uses `cmd /c start` to open the browser. If the default browser isn't properly registered, manually visit `http://localhost:5173/`.

### Music Search Returns No Results

1. First-time use requires clicking "Scan Music Library"
2. Scan scope includes `~/Music`, `~/Downloads`, `~/Desktop` and D/E/F drive music directories
3. Only scans up to 4 directory levels deep

### WebSocket Connection Status Abnormal

Connection status is driven by WebSocket `onopen` / `onclose` events, not message polling. If abnormal, check if the backend WebSocket service is running.
